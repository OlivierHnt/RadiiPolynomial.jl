Sequences

A Sequence is a structure representing a compactly supported element of sequence space.

Arithmetic

The arithmetic operations +,-,*,^ are implemented along with the convenient bar operations +̄,-̄,*̄,^̄ (+\bar<TAB>, -\bar<TAB>, *\bar<TAB>, ^\bar<TAB>) which are equivalent to

+̄(a::Sequence...) = embed(+(a...), mapreduce(aᵢ -> aᵢ.space, ∪̄, a))
-̄(a::Sequence...) = embed(-(a...), mapreduce(aᵢ -> aᵢ.space, ∪̄, a))
*̄(a::Sequence...) = embed(*(a...), mapreduce(aᵢ -> aᵢ.space, ∪̄, a))
^̄(a::Sequence, n::Int) = embed(^(a, n), a.space)
Note

Divisions and other elementary functions (e.g. exp,log,cos,sin) are purposely not supported as, in general, they yield sequences which are not compactly supported.

The internal routines of the multiplication are based on the fast Fourier transform (FFT) which needed to be re-implemented to ensure compatibility with IntervalArithmetic.jl. It is important to keep in mind that the interval representation widens severely with complex number multiplication which conflicts with the FFT algorithm. This has been taken into account by implementing *(a::Sequence...) such that we only perform the FFT aglorithm twice.

julia> using RadiiPolynomial, IntervalArithmetic
ERROR: ArgumentError: Package IntervalArithmetic not found in current path:
- Run `import Pkg; Pkg.add("IntervalArithmetic")` to install the IntervalArithmetic package.

julia> space = Taylor(2^12-1)
Taylor(4095)

julia> a = Sequence(space, [@interval(x) for x ∈ rand(length(space))])
ERROR: LoadError: UndefVarError: @interval not defined
in expression starting at none:1

julia> maximum(radius.(a))
ERROR: UndefVarError: radius not defined

julia> maximum(radius.(a^3))
ERROR: UndefVarError: a not defined

julia> maximum(radius.(a^5))
ERROR: UndefVarError: a not defined

The above shows a considerable loss of accuracy. Although, in practice series exhibit a decay which prevents such a large error to occur. Indeed, if we add a small decay to our series:

julia> using RadiiPolynomial, IntervalArithmetic
ERROR: ArgumentError: Package IntervalArithmetic not found in current path:
- Run `import Pkg; Pkg.add("IntervalArithmetic")` to install the IntervalArithmetic package.

julia> space = Taylor(2^12-1)
Taylor(4095)

julia> a = Sequence(space, [@interval(rand()/4.0^i) for i ∈ eachindex(space)])
ERROR: LoadError: UndefVarError: @interval not defined
in expression starting at none:1

julia> maximum(radius.(a^5))
ERROR: UndefVarError: a not defined
Note

The intervals precision can be adjusted via setprecision (cf. IntervalArithmetic.jl).

However, in Computing Discrete Convolutions with Verified Accuracy Via Banach Algebras and the FFT the author shows how to rigorously bound convolutions to avoid machine precision limitations. The algorithm can be used via the banach_algebra_rounding! function.

julia> using RadiiPolynomial, IntervalArithmetic
ERROR: ArgumentError: Package IntervalArithmetic not found in current path:
- Run `import Pkg; Pkg.add("IntervalArithmetic")` to install the IntervalArithmetic package.

julia> space = Taylor(2^4-1);

julia> a = Sequence(space, [@interval(rand()/4.0^i) for i ∈ eachindex(space)]);
ERROR: LoadError: UndefVarError: @interval not defined
in expression starting at none:1

julia> C = norm(a, 4.0)^3;
ERROR: UndefVarError: a not defined

julia> ν = [@interval(C/4.0^i) for i ∈ 0:3order(space)];
ERROR: LoadError: UndefVarError: @interval not defined
in expression starting at none:1

julia> a³ = a^3;
ERROR: UndefVarError: a not defined

julia> b = banach_algebra_rounding!(a^3, ν);
ERROR: UndefVarError: a not defined
